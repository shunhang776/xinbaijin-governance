# ADR-0001：治理核心采用成熟开源组件

## 状态

Accepted（候选实现，尚未进入生产门禁）

## 决策

- JSON Schema 2020-12 + Ajv 负责数据结构验证；
- OPA/Rego + Conftest 负责跨字段政策；
- Vitest + fast-check + Stryker 负责示例、属性和变异测试；
- GitHub Actions + Rulesets 负责执行和强制；
- 白槿只自研仓库映射、证据转换、回读完整性核验和业务政策。

## 不采用

- 不继续扩张手写 `trusted-core.js`；
- 不自研 JSON Schema 验证器、政策引擎、状态机引擎或 CI；
- 第一阶段不引入 XState、SQLite、Codex 扩展、Grafana 或 GitHub AI 审查。

## 风险

Rego 政策和证据转换仍属于白槿业务代码，因此必须经过政策测试、属性测试、变异测试、独立审查和影子运行。
